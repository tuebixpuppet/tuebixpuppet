Die Lösung: Zentrale Konfigurationsverwaltung
Puppetmaster
Die Konfigurationen werden hier zentral definiert.
Jeder Server erhält von hier eine Beschreibung der Konfigurationen die er durchführen muss.
Puppet-Agents
Jeder Puppet-Agent holt sich vom Puppetmaster die Beschreibung seiner durchzuführenden Konfigurationen, den Katalog.
Jeder Konfigurationsschritt aus dem Katalog wird vom Puppet-Agent durchgeführt.

Schritt 3: Directory Environments
Environments Jedem Puppet-Agent kann eine Environment zugeordnet werden z.B.: production, testing, ...
Directory Environments Sind eine elegante Art, verschiedenen Environments zu verwalten
Directory Environments Puppet-Agents können nach Environments gruppiert jeweils Konfigurationen aus unterschiedlichen Verzeichnisstrukturen erhalten.
